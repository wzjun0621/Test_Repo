
수행평가 1 프로세스 익히기: 
1. 선생님 깃허브 접속 -> Fork 클릭.
2. 본인 계정으로 온 저장소 주소 복사 -> Dev-C++에서 수정 후 파일 저장.
3. 깃허브 웹사이트에서 직접 파일 업로드(Upload files) 기능으로 수정본 반영하기.
4. Pull Request 버튼을 눌러 메시지 작성(예: [학번/이름] 수행평가 제출) 및 전송.

수행평가 1 가이드
새로운 예제 코드가 담긴 저장소 주소를 공개. 

학생들은 정해진 시간 내에 스스로 Fork -> 수정 및 주석 작성 -> PR까지 완료

선생님은 깃허브의 Pull Requests 탭에서 실시간으로 제출 현황을 확인하고 코드를 리뷰하여서 수행평가 평가를 할 예정.
